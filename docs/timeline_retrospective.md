# 🗓️ 슈킹(SHOOKING) 프로젝트 타임라인 및 기술 회고

## 📌 프로젝트 개요

슈킹(SHOOKING) 프로젝트는 단순한 쇼핑몰 UI 구현을 넘어, **확장 가능한 아키텍처**와 **실시간 데이터 연동**을 목표로 개발된 프리미엄 신발 커머스 플랫폼입니다. 로컬 목업 데이터에서 시작하여 클라우드 백엔드(Supabase) 연동에 이르기까지의 기술적 성장 과정을 기록합니다.

---

## 🧱 기술적 진화 (Architectural Evolution)

| 단계 | 주요 변화 | 핵심 기술 |
| :--- | :--- | :--- |
| **Phase 1: 기반 구축** | UI 컴포넌트 설계 및 정적 데이터 렌더링 | React 18, CSS Modules |
| **Phase 2: 상태 관리 최적화** | Prop Drilling 해결 및 전역 상태 도입 | Context API (Cart, User, Theme) |
| **Phase 3: 백엔드 통합** | 실시간 DB 연동 및 비동기 데이터 관리 | Supabase, React Query |
| **Phase 4: 사용자 경험 고도화** | 성능 최적화, 애니메이션, 결제 및 챗봇 | Framer Motion, Lazy Loading, PaymentModal |

---

## 📅 타임라인 및 주요 마일스톤

### 1️⃣ 요구사항 정의 및 MVP 개발 (Week 1-2)
- **목표**: 2컬럼 그리드 레이아웃의 상품 목록 및 장바구니 기본 기능 구현.
- **성과**: 모바일 퍼스트 디자인 적용, `ProductCard` 컴포넌트화.
- **인사이트**: 초기 컴포넌트 분리 수준이 향후 재사용성을 결정함을 체감.

### 2️⃣ 데이터 아키텍처 고도화 (Week 3-4)
- **목표**: 로컬 `products.js` 의존성 탈피 및 Supabase 연동.
- **성과**: `productsApi.js` 구축, `React Query`를 통한 서버 상태(Server State) 관리 도입.
- **인사이트**: `useEffect` 기반의 fetch보다 `useQuery`가 제공하는 캐싱과 로딩 상태 관리가 DX(개발자 경험)를 비약적으로 상승시킴.

### 3️⃣ 복합 기능 및 서비스 안정화 (Week 5-6)
- **목표**: 결제 프로세스, 관리자 대시보드, 고객 지원 챗봇 구현.
- **성과**: `PaymentModal` 통합, `SupportBot` 독립 컴포넌트 구현, `ErrorBoundary` 적용.
- **인사이트**: 사용자 실수나 네트워크 에러에 대응하는 'Graceful Degradation(우아한 성능 저하)'의 중요성 학습.

## 🔍 핵심 기술 회고 (Deep Dive)

### ✅ 잘한 점 (Successes)
1. **Context API의 전략적 활용**: 장바구니, 결제, 테마 등 도메인별로 Context를 분리하여 단일 책임 원칙(SRP)을 준수함.
2. **React Query 도입**: 데이터 페칭 로직을 선언적으로 작성하여 컴포넌트 코드를 간결하게 유지함.
3. **사용자 인터랙션 강화**: `Framer Motion`을 활용한 페이지 전환 및 모달 애니메이션으로 앱의 완성도를 높임.
4. **테스트 주도 개발 시도**: `cartUtils.test.js` 등 핵심 로직에 대한 단위 테스트를 작성하여 안정성 확보.

### ⚠️ 어려웠던 점 및 해결 (Challenges & Solutions)
- **문제**: 복잡한 라우팅 구조에서 페이지 전환 시 상태 초기화 및 애니메이션 충돌 발생.
- **해결**: `react-router-dom` v7의 최신 기능을 활용하고, `AnimatePresence`의 `mode="wait"` 옵션을 통해 시퀀스를 제어함.
- **문제**: Supabase 실시간 구독 시 메모리 누수 위험.
- **해결**: `useEffect` 내에서 cleanup 함수를 정확히 구현하여 구독을 해제하는 패턴을 정착시킴.

## 🔧 향후 개선 과제 (Action Plan)

1. **사용자 인증 고도화**: 현재의 간단한 Login 방식을 Supabase Auth(OAuth, 이메일 인증)로 전환.
2. **이미지 최적화**: WebP 포맷 도입 및 Cloudinary/Supabase Storage CDN 활용을 통한 로딩 속도 극대화.
3. **PWA(Progressive Web App) 도입**: 모바일 사용자들을 위해 오프라인 모드 및 푸시 알림 기능 추가.
4. **E2E 테스트 강화**: Cypress 또는 Playwright를 도입하여 결제부터 주문 확인까지의 전 과정을 자동화 테스트.
