# 03. 장바구니(Cart) 요구사항 사양서

## 1. 개요
상품 옵션(사이즈)별 분리 관리와 연관 상품 추천(Upselling)이 결합된 장바구니 시스템.

## 2. 기능 요구사항

### 2.1 상품 관리
- **사이즈별 분리**: 동일 상품이라도 사이즈가 다르면 개별 항목으로 관리 (ID-Size Key).
- **수량 조절**: 1~99개 실시간 수량 변경 및 금액 즉시 반영.
- **삭제 기능**: 개별 항목 삭제 및 장바구니 전체 비우기 지원.
- **데이터 유지**: 비로그인 시 `localStorage`, 로그인 시 프로필 기반 데이터 동기화(향후 확장).

### 2.2 자동 계산 로직
- **배송비 정책**: 50,000원 이상 구매 시 무료 배송 (미만 시 3,000원 자동 추가).
- **실시간 합계**: 상품 가격 * 수량 + 배송비 합계 시각화.

### 2.3 매출 고도화 (Upselling)
- **액세서리 추천**: 양말, 세척 키트 등 신발 관련 용품 하단 상시 노출 및 간편 추가 버튼 제공.

## 3. UI/UX 명세
- **토스트 알림**: 담기/삭제 성공 시 화면 중앙 하단 애니메이션 알림 제공.
- **슬라이드 효과**: 항목 삭제 시 리스트 레이아웃의 자연스러운 전환.
- **고정 하단 결제바**: 모바일 환경을 고려한 하단 총액 및 결제 버튼 배치.
