## 2. 디자인 시안 구성 요소

피그마 시안(`node-id: 132-969`) 기준 장바구니 페이지의 UI 구성입니다.

### 2-1. 헤더 영역

- **배경색**: 검정(#000000)
- **뒤로가기 버튼(←)**: 좌측 상단 흰색 화살표 아이콘
- 타이틀 텍스트 없이 아이콘만 표시

### 2-2. 페이지 타이틀 영역

- **"장바구니"** 타이틀 (굵은 텍스트, 대형)
- 부제: `현재 3개의 상품이 담겨있습니다.` (회색 소형 텍스트)

### 2-3. 상품 목록 영역

시안 기준 3개의 상품이 세로로 나열됩니다.

| 상품 | 브랜드 | 가격 | 수량 |
|------|--------|------|------|
| 흰색 Nike 운동화 | 브랜드A | 35,000원 | 1 |
| 파스텔 컬러 운동화 | 브랜드A | 25,000원 | 1 |
| 빨간 Nike 운동화 | 브랜드B | 35,000원 | 1 |

각 상품 카드 구성:
- **상품 이미지**: 좌측 정사각형 썸네일 (둥근 모서리)
- **브랜드명**: 우측 상단 (회색 소형 텍스트)
- **가격**: 브랜드명 아래 (굵은 대형 텍스트, `원` 단위)
- **수량 조절**: `−  1  +` 버튼 (원형 버튼)

### 2-4. 주문 요약 영역

구분선 위에 금액 정보가 표시됩니다.

| 항목 | 금액 |
|------|------|
| 상품 금액 | 95,000원 |
| 배송비 | 3,000원 (10만원 미만이므로 유료) |
| **총 금액** | **98,000원** |

- 상품 금액과 배송비는 일반 텍스트
- **총 금액**은 굵은 텍스트로 강조
- 배송비 정책: 10만원 미만 → 3,000원 / 10만원 이상 → 무료

### 2-5. 결제 버튼 영역

- **"결제하기"** 버튼: 전체 너비, 검정 배경, 흰색 텍스트
- 둥근 모서리(border-radius 큰 값)
- 페이지 하단 고정

## 3. 구현 vs 시안 비교 체크리스트

> 체크 기준: ✅ 구현 완료 / ⚠️ 일부 미흡 / ❌ 미구현

### 레이아웃

| 항목 | 상태 | 비고 |
|------|------|------|
| 검정 헤더 + 흰색 뒤로가기(←) 버튼 | ✅ | |
| "장바구니" 타이틀 + 상품 수 부제 | ✅ | |
| 상품 카드 세로 리스트 (이미지 + 브랜드 + 가격 + 수량) | ✅ | |
| 수량 조절 버튼 (`−` / `+`) | ✅ | |
| 주문 요약 영역 (상품 금액 / 배송비 / 총 금액) | ✅ | |
| 결제하기 버튼 (전체 너비, 하단 고정) | ✅ | |

### 기능

| 항목 | 상태 | 비고 |
|------|------|------|
| 상품별 소계 금액 계산 (단가 × 수량) | ✅ | |
| 전체 상품 합계 자동 계산 | ✅ | 시안 기준 95,000원 |
| 배송비 자동 계산 (10만원 기준) | ✅ | 시안: 95,000원 → 3,000원 |
| 총 금액 계산 (상품 금액 + 배송비) | ✅ | 시안 기준 98,000원 |
| 뒤로가기(←) 버튼 → 메인 상품 목록 이동 | ✅ | |
| 결제하기 버튼 클릭 → 결제 창 노출 | ✅ | |
| 카드 등록 및 결제 처리 | ✅ | |

### 스타일

| 항목 | 상태 | 비고 |
|------|------|------|
| 헤더 배경색 검정 | ✅ | |
| 상품 이미지 둥근 모서리 | ✅ | |
| 수량 버튼 원형 스타일 | ✅ | |
| 결제 버튼 검정 + 흰 텍스트 + 둥근 모서리 | ✅ | |
| 총 금액 굵은 텍스트 강조 | ✅ | |
| 모바일 기준 430px 레이아웃 | ✅ | |

## 4. 기능 테스트 결과

### 테스트 환경

- 브라우저: Chrome 최신 버전
- 해상도: 430px (피그마 시안 기준 모바일)
- 접속 URL: `http://localhost:3000/cart`

### 테스트 항목

| No. | 테스트 항목 | 예상 결과 | 실제 결과 | 상태 |
|-----|-----------|---------|---------|------|

| 2 | 배송비 (총액 10만원 미만) | **3,000원** 표시 | 3,000원 표시 | ✅ PASS |

| 6 | 결제하기 버튼 클릭 | 결제 창(모달/페이지) 노출 | 결제창 노출 | ✅ PASS |

## 5. 이슈 및 개선 사항

### 발견된 이슈

| No. | 이슈 내용 | 심각도 | 상태 |
|-----|----------|--------|------|
| 1 | 소형 모바일(320px 이하)에서 결제 모달 레이아웃 깨짐 | 낮음 | 개선 예정 |
| 2 | 새로고침 시 장바구니 데이터 초기화 가능성 | 중간 | 확인 필요 |
| 3 | 수량 0 설정 시 동작 미정의 (자동 삭제 여부 불명확) | 낮음 | 확인 필요 |

### 개선 제안

- 장바구니가 비어 있을 때 **빈 장바구니 안내 UI** 추가 권장 (시안에 미포함)
- 수량 `−` 버튼으로 0이 됐을 때 상품 자동 삭제 또는 삭제 확인 알림 처리 필요
- 시안의 부제(`현재 3개의 상품이 담겨있습니다.`)는 장바구니 상품 수에 따라 **동적으로 변경** 처리 필요
