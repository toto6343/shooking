📝 결제 모듈 리뷰 (Payment Module Review)

SHOOKING 결제 모듈은 장바구니를 생략한 즉시 결제(One-step Payment) 흐름을 중심으로 설계된 프론트엔드 결제 컴포넌트입니다.
사용자가 결제 과정에서 느낄 수 있는 입력 피로도와 불확실성을 최소화하기 위해, 결제 단계를 명확히 분리하고 모달 기반 UI를 적용했습니다.

특히 카드 등록 → 카드 선택 → 결제 확인 → 결제 결과 안내의 흐름을 하나의 일관된 UX로 연결하여,
실제 커머스 서비스에서 바로 확장 가능한 결제 모듈 구조를 목표로 구현했습니다.

📌 고객사 요구사항 정리
1. 이메일 기반 핵심 요구사항

장바구니 없이 즉시 결제가 가능한 결제 플로우 제공

카드 등록 후 여러 카드 중 선택하여 결제 가능

카드 정보 노출을 최소화한 보안 친화적 UI

결제 성공 / 실패에 대한 명확한 사용자 안내

모달 기반 결제 UI 제공

2. Figma 디자인 시안 기반 추가 요구사항

카드 번호 입력 시 자동 포맷팅 및 마스킹 처리

카드 유효기간(MM/YY) 입력 시 숫자 외 입력 제한

CVC 입력 필드 비노출(마스킹) 처리

선택된 카드에 대한 시각적 강조(Active 상태)

결제 전 최종 확인 모달 제공

결제 처리 중 로딩 상태 표시

오류 발생 시 원인에 대한 텍스트 안내 제공

✅ 요구사항에 따른 기능 동작 확인
1. 카드 등록 기능

카드 번호, 유효기간, CVC 입력 필드를 제공

입력 값에 대한 실시간 유효성 검사 수행

카드 번호는 마스킹 처리되어 화면에 노출

유효하지 않은 입력 시 경고 메시지 표시

✔️ 요구사항 충족 여부: 충족

2. 카드 목록 및 카드 선택 기능

등록된 카드 목록을 리스트 형태로 제공

사용자가 결제에 사용할 카드 선택 가능

선택된 카드는 시각적으로 강조 표시

카드 변경 시 결제 대상 카드 즉시 반영

✔️ 요구사항 충족 여부: 충족

3. 결제 확인 모달

결제 버튼 클릭 시 결제 확인 모달 표시

선택한 카드 정보 및 결제 금액 확인 가능

결제 진행 / 취소 선택 가능

결제 진행 중 로딩 상태 표시

✔️ 요구사항 충족 여부: 충족

4. 결제 성공 / 실패 처리

Mock 결제 로직을 통한 결제 결과 분기 처리

결제 성공 시 성공 메시지 및 완료 상태 표시

결제 실패 시 실패 메시지 및 재시도 안내 제공

결제 결과에 따라 모달 상태 변경

✔️ 요구사항 충족 여부: 충족 (Mock 기반)

5. 상태 관리 구조

Context API를 활용하여 카드 정보 및 결제 상태 관리

컴포넌트 간 불필요한 props 전달 최소화

결제 흐름 전반의 상태 일관성 유지

✔️ 요구사항 충족 여부: 충족

🔍 개선 포인트 및 향후 확장 방향

실제 결제 API(PG사) 연동을 통한 실결제 환경 대응

결제 실패 사유(한도 초과, 인증 실패 등) 세분화 처리

모바일 환경을 고려한 반응형 UI 개선

접근성(A11y)을 고려한 키보드 조작 및 ARIA 속성 추가

결제 모듈을 패키지화하여 외부 프로젝트에서 재사용 가능하도록 구조 개선
