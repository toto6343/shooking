# 🛒 슈킹(SHOOKING) 쇼핑몰 연동 프로젝트 가이드

## 1. 프로젝트 개요
본 프로젝트는 2030 세대를 위한 프리미엄 신발 전문 쇼핑몰 '슈킹(SHOOKING)'의 프론트엔드 시스템입니다. 실제 상용 서비스 수준의 사용자 경험을 제공하기 위해 Supabase 백엔드 연동 및 다양한 인터랙티브 기능을 포함하고 있습니다.

### 🎯 프로젝트 목적
- 실시간 데이터베이스(Supabase) 연동을 통한 동적 상품 관리
- React Query를 활용한 효율적인 데이터 페칭 및 캐싱 전략 수립
- 사용자 중심의 직관적인 쇼핑 UI/UX 구현 (장바구니, 찜하기, 결제 흐름)

- **로컬 개발 서버**: [http://localhost:5173](http://localhost:5173)
- **배포 URL**: (배포 시 실제 URL로 업데이트 예정)

## 4. 프로젝트 구조 및 컴포넌트 역할

### 📂 주요 폴더 구조
- `src/api`: Supabase 클라이언트 및 데이터 통신 로직
- `src/components`: UI 컴포넌트 (Header, ProductCard, Cart 등)
- `src/context`: 도메인별 전역 상태 관리
- `src/utils`: 데이터 가공 및 단위 테스트 코드

### 📦 주요 컴포넌트 설명
- **Header**: 로고, 검색, 장바구니 실시간 카운트 표시 및 다크모드 토글
- **ProductList**: 메인 상품 목록을 2컬럼 그리드로 렌더링
- **ProductDetail**: 상품 상세 정보(설명, 가격, 리뷰 등) 제공 및 옵션 선택
- **Cart**: 장바구니에 담긴 상품 목록 조회, 수량 조절 및 삭제
- **PaymentModal**: 통합 결제 시스템 인터페이스
- **SupportBot**: 실시간 고객 지원 챗봇

## 5. 기능 동작 및 테스트 시나리오
아래 항목들은 실제 서비스에서 정상 동작함을 확인한 핵심 기능입니다.

1.  **상품 상세 접속**: 메인 페이지에서 제품 카드를 클릭하면 `/product/:id` 경로의 상세 페이지로 즉시 이동합니다.
2.  **장바구니 실시간 카운트**: 제품 카드나 상세 페이지에서 '담기' 버튼을 누르면 헤더의 장바구니 아이콘 숫자가 실시간으로 증가합니다.
3.  **결제 프로세스**: 장바구니에서 '결제하기' 버튼을 누르면 결제 모달이 뜨며, 결제 성공 시 주문 완료 페이지(`/order-success`)로 이동합니다.
4.  **수량 유지 및 반영**: 상세 페이지나 장바구니에서 수량을 조절한 후 다른 페이지로 이동했다가 돌아와도 수량 및 총 합계 금액이 정확히 유지됩니다. (Context API 기반 상태 보존)

## 7. 유의사항 및 알려진 이슈
- **Supabase 연결**: `.env` 파일에 유효한 API Key가 설정되어 있어야 상품 데이터가 정상적으로 표시됩니다.
- **새로고침 시 상태**: 현재 장바구니 상태는 메모리(Context)에 저장되므로, 새로고침 시 초기화될 수 있습니다. (향후 LocalStorage 연동 예정)
- **이미지 로딩**: 고해상도 이미지 사용 시 네트워크 속도에 따라 로딩 지연이 발생할 수 있습니다.
