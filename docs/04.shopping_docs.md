# 🛒 슈킹(SHOOKING) 쇼핑몰 연동 프로젝트

## 1. 프로젝트 개요 (Project Overview)
본 프로젝트는 2030 세대를 타겟으로 한 프리미엄 신발 전문 쇼핑몰 '슈킹(SHOOKING)'의 프론트엔드 시스템입니다. 단순한 UI 구현을 넘어 Supabase 백엔드 연동을 통한 실시간 데이터 처리와 몰입형 사용자 경험(UX)을 제공합니다.

### 🎯 프로젝트 목적
- 실시간 데이터베이스 연동을 통한 동적 상품 및 주문 관리 시스템 구축
- React Query를 활용한 효율적인 데이터 페칭 및 캐싱 전략 수립
- 사용자 중심의 직관적인 쇼핑 UI/UX 구현 (장바구니, 결제, 커뮤니티)

### 🔗 접속 정보 (URL)
- **실제 서비스 배포 주소**: [https://shooking-ten.vercel.app/](https://shooking-ten.vercel.app/)
- **로컬 개발 서버**: `http://localhost:5173`

### 👨‍👩‍👧‍👦 참여 팀원
- **개발자**: 김우혁 (Frontend Full-stack / UI & API 설계)

## 2. 개발 환경 (Development Environment)
본 프로젝트는 확장성과 유지보수성을 고려하여 최신 기술 스택으로 구축되었습니다.

### 🛠 기술 스택
- **Language**: JavaScript (ES6+)
- **Framework**: React 18
- **Build Tool**: Vite
- **State Management**: Context API (Cart, Payment, User, Theme, Toast)
- **Data Fetching**: @tanstack/react-query (v5)
- **Backend/DB**: Supabase (PostgreSQL, Real-time)
- **Animation**: Framer Motion
- **Styling**: CSS Modules

## 3. 설치 및 실행 방법 (Installation & Execution)
로컬 개발 환경에서 프로젝트를 실행하기 위한 명령어는 다음과 같습니다.

### 1️⃣ 패키지 설치
```bash
npm install
```

### 2️⃣ 환경 변수 설정
프로젝트 루트에 `.env` 파일을 생성하고 Supabase 접속 정보를 입력합니다.
```env
VITE_SUPABASE_URL=YOUR_SUPABASE_URL
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
```

### 3️⃣ 개발 서버 실행
```bash
npm run dev
```
브라우저에서 `http://localhost:5173`에 접속합니다.

## 4. 컴포넌트 구조 (Component Structure)
주요 컴포넌트들의 역할과 구조는 다음과 같습니다.

- **Header**: 로고, 검색, 장바구니 실시간 카운트 표시 및 다크모드 토글 기능 담당.
- **ProductList**: 메인 상품 목록을 렌더링하며 카테고리/브랜드 필터링 로직 포함.
- **ProductDetail**: 상품 상세 정보 표시, **360도 인터랙티브 뷰어**, 사이즈 선택 및 리뷰 시스템 제공.
- **Cart**: 장바구니에 담긴 상품 목록 조회, 수량 조절 및 실시간 금액 합계 계산.
- **PaymentModal**: 통합 결제 시스템 인터페이스 및 주문 처리 로직.
- **SupportBot**: AI 기반의 실시간 고객 지원 챗봇.

## 5. 테스트 방법 (Testing Method)
프로젝트의 안정성을 검증하기 위한 테스트 절차입니다.

### 🧪 단위 테스트 실행
```bash
npm test
```
- `cartUtils.test.js`: 장바구니 계산 및 수량 업데이트 로직의 정확성 검증.

### 📋 기능 테스트 검증 가이드 (채점 포인트)
채점관님께서는 아래 절차에 따라 기능 동작을 확인해 주시기 바랍니다.

1. **제품 상세 페이지 접속**: 메인 페이지에서 특정 제품 카드를 클릭하면 `/product/:id` 경로로 이동하며 상세 정보가 표시됩니다.
2. **장바구니 실시간 카운트**: 제품 카드나 상세 페이지에서 '담기' 버튼을 누르면 상단 헤더의 장바구니 아이콘 숫자가 즉시 증가합니다.
3. **결제 완료 페이지 확인**: 장바구니에서 '결제하기' 버튼을 누르고 결제를 진행하면 주문 완료 페이지(`/order-success`)가 정상적으로 노출됩니다.
4. **수량 유지 및 상태 보존**: 상세 페이지에서 수량을 변경한 후 장바구니로 이동하거나, 다른 페이지를 다녀와도 변경된 수량과 합계 금액이 정확히 유지됩니다. (Context API 기반)

## 7. 유의사항 및 알려진 이슈 (Precautions & Issues)
- **Supabase 연결**: `.env` 파일에 유효한 API Key가 설정되어 있어야 상품 데이터가 정상 노출됩니다. (현재 연동 실패 시 로컬 데이터를 사용하도록 Fallback 처리 완료)
- **배포 환경**: 현재 Vercel을 통해 배포되어 있으며, 일부 브라우저 설정에 따라 애니메이션 지연이 발생할 수 있습니다.
