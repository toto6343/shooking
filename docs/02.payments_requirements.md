## 📝 고객 요구사항

### 1. 카드 등록 관련 요구사항

- 사용자는 여러 개의 결제 카드를 등록할 수 있어야 한다.
- 카드 등록 과정에서 카드 번호, 유효기간, CVC 등 민감한 정보가 화면에 그대로 노출되지 않아야 한다.
- 카드 번호 입력 시 가독성과 보안을 고려한 UI가 필요하다.
- 등록된 카드는 결제 시 선택 가능해야 한다.

### 2. 결제 동작 관련 요구사항

- 상품 목록 화면에서 기존 '담기' 버튼 옆에 '결제' 버튼을 추가한다.
- 사용자는 장바구니를 거치지 않고도 즉시 결제할 수 있어야 한다.
- 결제 버튼 클릭 시 **카드 선택 → 결제 확인 → 결제 완료** 플로우가 자연스럽게 이어져야 한다.

### 3. UX / UI 요구사항

- Figma 디자인 시안을 기준으로 UI를 구현한다.
- 입력 오류 시 사용자에게 명확한 피드백을 제공한다.
- 결제 진행 중임을 알 수 있는 로딩 또는 상태 표시가 필요하다.

## 📋 기능 목록

### 1️⃣ 카드 정보 입력 컴포넌트

**카드 번호 입력**
- 카드 번호를 4자리 단위로 구분하여 입력
- 입력 시 자동 포커스 이동
- 숫자 외 입력 제한
- 화면 표시 시 일부 마스킹 처리 (예: `**** **** **** 1234`)

**카드 유효기간 입력**
- MM / YY 형식으로 입력
- 잘못된 월/연도 입력 시 즉시 오류 표시

**CVC 입력**
- 숫자만 입력 가능
- 입력 값 마스킹 처리

---

### 2️⃣ 카드 관리 기능

- 등록된 카드 목록 조회
- 기본 결제 카드 표시
- 카드 선택 기능
- 카드 삭제 기능

---

### 3️⃣ 결제 기능

- 상품 목록 페이지에서 바로 결제 가능
- 카드 선택 후 결제 진행
- 결제 진행 중 로딩 상태 표시
- 결제 성공 / 실패 결과 처리

---

### 4️⃣ 결제 플로우 관리

- 결제 확인용 모달 UI
- 결제 금액, 상품 정보 요약 표시
- 결제 취소 / 확인 버튼 제공

---

### 5️⃣ 예외 및 오류 처리

- 카드 정보 미입력 시 결제 불가
- 잘못된 카드 정보 입력 시 오류 메시지 표시
- 결제 실패 시 사용자에게 재시도 안내

## 🔄 결제 플로우

```
상품 목록 페이지
      │
      ▼
  [결제] 버튼 클릭
      │
      ▼
  카드 선택
      │
      ▼
  결제 확인 모달
  (상품 정보 + 결제 금액 요약)
      │
    ┌─┴─┐
  취소  확인
    │     │
    ▼     ▼
  닫기  결제 진행 중 (로딩)
            │
          ┌─┴─┐
        성공  실패
          │     │
          ▼     ▼
       완료 화면  오류 안내 + 재시도
```
