# 02. 결제(Payments) 요구사항 사양서

## 1. 개요
보안성이 강화된 카드 결제와 다양한 간편결제 수단, 그리고 선물하기 기능을 통합한 결제 시스템.

## 2. 기능 요구사항

### 2.1 결제 프로세스
- **Supabase Auth 연동**: 로그인된 사용자 정보를 기반으로 결제 진행 및 포인트 적립.
- **배송지 관리**: 우편번호 검색(Daum API) 연동 및 기본 배송지 자동 저장.
- **선물하기 기능**: 받는 사람의 연락처와 메시지만으로 결제 가능한 별도 프로세스 지원.
- **쿠폰 시스템**: 할인 코드(예: WELCOME30) 적용 시 실시간 금액 할인 계산.
- **멤버십 혜택**: 등급별(Silver 5%, Gold 10%) 자동 추가 할인 적용.

### 2.2 결제 수단
- **카드 결제**: 카드 번호(16자리), 유효기간, CVC, 비밀번호 실시간 검증 및 마스킹.
- **간편 결제**: 카카오페이, 네이버페이, 토스페이 UI 연동 및 모사.

### 2.3 사후 처리
- **주문 내역 저장**: Supabase `orders` 테이블에 상세 정보 영구 저장.
- **포인트 적립**: 결제 금액의 1% 자동 적립 및 프로필 업데이트.
- **주문 성공 페이지**: 주문 번호 및 배송 예정일 정보를 포함한 전용 결과 UI 노출.

## 3. UI/UX 명세
- **인터랙티브 모달**: 4단계(배송/선물 -> 결제수단 -> 확인 -> 처리) 진행 바가 포함된 모달.
- **카드 미리보기**: 정보 입력 시 가상 카드 이미지에 데이터 실시간 동기화.
- **애니메이션 스피너**: 결제 처리 중 시각적 대기 피드백 제공.
