# 장바구니 페이지 문서화 (Cart Page Documentation)

---

## 1. 프로젝트 개요

장바구니 페이지는 사용자가 선택한 상품을 확인하고 결제를 진행할 수 있는 페이지입니다.  
사용자는 장바구니에 담긴 상품 목록을 확인하고 총 금액을 계산한 뒤 결제할 수 있습니다.

### 주요 기능

- 장바구니에 담긴 상품 목록 확인
- 상품 가격 합계 계산
- 배송비 자동 계산 (10만원 미만: 3,000원 / 10만원 이상: 무료)
- 결제하기 기능 (카드 등록 및 결제)
- 뒤로가기 버튼을 통해 상품 목록 페이지로 이동

### 페이지 URL

```
http://localhost:3000/cart
```

### 참여한 팀원

| 이름 | 역할 |
|------|------|
| 김우혁 | 장바구니 페이지 개발 |

## 3. 설치 및 실행 방법

### 1️⃣ 프로젝트 클론

```bash
git clone https://github.com/toto6343/shooking.git
cd shooking
```

### 2️⃣ 패키지 설치



## 4. 컴포넌트 구조

```
src/
├── pages/
│   └── CartPage.jsx          # 장바구니 페이지 최상위 컴포넌트
├── components/
│   ├── CartItem.jsx           # 개별 장바구니 상품 카드
│   ├── CartSummary.jsx        # 주문 금액 요약 (소계, 배송비, 합계)
│   └── CheckoutButton.jsx     # 결제하기 버튼
└── utils/
    └── cartUtils.js           # 금액 계산, 배송비 계산 로직
```

### 주요 컴포넌트 설명

#### `CartPage.jsx`
- 장바구니 페이지의 루트 컴포넌트
- 상품 목록과 주문 요약을 렌더링
- 뒤로가기 버튼으로 메인 상품 목록 페이지(`/`) 이동 처리

#### `CartItem.jsx`
- 장바구니에 담긴 개별 상품 정보 표시 (이름, 가격, 수량, 이미지)
- 상품별 소계 금액 계산 및 표시

#### `CartSummary.jsx`
- 전체 상품 금액 합산
- 배송비 계산 로직:
  - 총 금액 **10만원 미만** → 배송비 **3,000원**
  - 총 금액 **10만원 이상** → 배송비 **무료**
- 최종 결제 금액 표시

#### `CheckoutButton.jsx`
- 결제하기 버튼 컴포넌트
- 클릭 시 결제 모달 또는 결제 페이지로 이동
- 카드 등록 및 결제 처리

## 5. 테스트 방법 및 스토리북

### 단위 테스트 실행

```bash
npm test
```

특정 컴포넌트만 테스트:

```bash
npm test -- --testPathPattern=CartItem
```

테스트 커버리지 확인:

```bash
npm test -- --coverage
```

### 테스트 주요 항목

| 테스트 항목 | 설명 |
|------------|------|
| 금액 합계 계산 | 장바구니 상품들의 총금액이 정확히 계산되는지 확인 |
| 배송비 계산 | 10만원 기준으로 배송비가 올바르게 적용되는지 확인 |
| 뒤로가기 라우팅 | 뒤로가기 버튼 클릭 시 `/` 경로로 이동하는지 확인 |
| 결제 버튼 동작 | 결제하기 버튼 클릭 시 결제 창이 노출되는지 확인 |

### 스토리북 실행

```bash
npm run storybook
```

실행 후 브라우저에서 [http://localhost:6006](http://localhost:6006) 접속

> 스토리북에서 `CartItem`, `CartSummary`, `CheckoutButton` 컴포넌트를 개별적으로 확인할 수 있습니다.

## 6. 유의사항 및 알려진 이슈

### 유의사항

- 장바구니 데이터는 **localStorage** 또는 **전역 상태(Context/Redux)** 에 저장됩니다. 브라우저 초기화 시 데이터가 소실될 수 있습니다.
- 결제 기능은 실제 PG사와 연동된 기능으로, **테스트 환경에서는 실결제가 발생하지 않도록** 테스트 모드로 실행해야 합니다.
- 배송비 정책(10만원 기준)은 `cartUtils.js` 내 상수 값으로 관리되며, 정책 변경 시 해당 파일을 수정해야 합니다.

### 알려진 이슈

| 이슈 | 설명 | 상태 |
|------|------|------|
| 새로고침 시 장바구니 초기화 | 상태 관리가 메모리에만 저장될 경우 새로고침 시 장바구니가 초기화될 수 있음 | 확인 필요 |
| 모바일 결제 UI | 일부 소형 디바이스에서 결제 모달 레이아웃이 깨질 수 있음 | 개선 예정 |
| 수량 0 처리 | 상품 수량을 0으로 설정 시 자동 삭제 여부 미정의 | 확인 필요 |
