# SHOOKING – 상품 목록 페이지 리뷰 및 요구사항 정리

## 📌 프로젝트 개요
SHOOKING은 젊은 세대를 타겟으로 한 신발 쇼핑몰 서비스입니다.  
본 문서는 **상품 목록 페이지(Product List Page)**를 기준으로 고객사 요구사항과  
요구사항에 따른 기능 동작 여부를 정리한 리뷰 문서입니다.

---

## 🧾 고객사 요구사항

### 1. 기본 요구사항 (이메일 기반)
1. 젊은 세대가 주로 **모바일 환경**을 사용하므로 모바일 사용성을 가장 중요하게 고려합니다.
2. 신발 상품의 **이미지가 디자인 시안(Figma)**처럼 잘 보이도록 구현해야 합니다.
3. 상품을 **장바구니에 추가할 경우**, 헤더 영역에서 **실시간으로 상품 개수가 표시**되어야 합니다.
4. 작업 결과를 확인할 수 있도록 **테스트용 배포 URL**을 제공합니다.

---

### 2. Figma 시안 기반 추가 요구사항
- 모바일 환경에서 상품 목록은 **2컬럼 그리드 레이아웃**을 유지합니다.
- 상품 이미지는 **일정한 비율(예: 1:1)**로 노출되어 목록의 정렬감을 유지합니다.
- 장바구니 담기 버튼은 클릭 시 **즉각적인 시각적 피드백**을 제공합니다.
- 다양한 모바일 해상도에서도 UI가 깨지지 않도록 **반응형 스타일**을 적용합니다.
- 상품 카드 간 간격, 텍스트 크기, 버튼 터치 영역은 모바일 UX 기준에 맞게 설계합니다.

---

## ✅ 요구사항에 따른 기능 동작 확인

### 1. 모바일 환경 구성
- [ ] 모바일 Safari에서 정상적으로 페이지가 로드됩니다.
- [ ] 삼성 브라우저에서 UI가 깨지지 않고 정상 동작합니다.
- [ ] 터치 기반 인터랙션(버튼 클릭, 스크롤)이 원활합니다.
- [ ] 모바일 화면 크기에 따라 반응형 레이아웃이 적용됩니다.

---

### 2. 신발 상품 사진
- [ ] 상품 이미지가 디자인 시안과 유사한 크기와 비율로 표시됩니다.
- [ ] 이미지가 깨지지 않고 선명하게 노출됩니다.
- [ ] 모든 상품 카드에서 이미지 비율이 동일하게 유지됩니다.
- [ ] 모바일 환경에서도 이미지 가독성이 확보됩니다.

---

### 3. 장바구니 담기 기능
- [ ] 상품 카드 내 장바구니 버튼 클릭 시 정상적으로 상품이 추가됩니다.
- [ ] 장바구니에 상품 추가 시 **헤더의 상품 개수가 실시간으로 증가**합니다.
- [ ] 버튼 클릭 시 색상/텍스트/아이콘 등 시각적 피드백이 제공됩니다.
- [ ] 동일 상품을 중복으로 담을 경우의 정책이 명확하게 동작합니다.

---

### 4. 배포 URL
- [ ] 테스트용 배포 URL이 제공됩니다.
- [ ] 배포 환경에서 모바일/데스크톱 모두 접근 가능합니다.
- [ ] 고객사에서 별도 설정 없이 기능 확인이 가능합니다.

> 🔗 **배포 URL**  
> (예: https://shooking.vercel.app)

## 🔍 추가 기능 및 품질 체크

### UI / UX
- [ ] 버튼 및 텍스트 크기가 모바일 환경에서 충분히 읽기 쉽습니다.
- [ ] 스크롤 시 화면 전환이 부드럽습니다.
- [ ] 헤더 UI가 사용자 경험을 방해하지 않습니다.

### 접근성
- [ ] 상품 이미지에 대체 텍스트(alt)가 포함되어 있습니다.
- [ ] 기본적인 웹 접근성 가이드를 준수합니다.

### 성능
- [ ] 이미지 로딩 시 사용자 경험을 저해하지 않습니다.
- [ ] 불필요한 렌더링 없이 페이지가 안정적으로 동작합니다.

### 브라우저 호환성
- [ ] Chrome / Safari / Samsung Browser 등 주요 브라우저에서 정상 동작합니다.
- [ ] 데스크톱 환경에서도 기본 기능이 정상 동작합니다.
