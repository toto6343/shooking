# 👟 SHOOKING (슈킹)

> 온라인 쇼핑몰 상품 정보를 기반으로 프론트엔드 UI와 백엔드 데이터를 연동하여 실제 서비스 형태로 구현하는 프로젝트

---

## 📌 프로젝트 개요

**SHOOKING**은 단순 UI 구현을 넘어, **데이터 흐름 중심의 개발 경험**과 **프론트엔드-백엔드 간 실제 협업 구조**를 직접 적용해보는 것을 목적으로 한 쇼핑몰 프로젝트입니다.

### 🎯 목표

- 쇼핑몰 상품 리스트 및 상세 페이지 구현
- API 연동을 통한 데이터 기반 UI 구성
- 사용자 친화적인 인터페이스 설계
- 실제 서비스 흐름과 유사한 구조 경험

### 🔍 해결하고자 한 문제

- 정적인 UI가 아닌 **동적인 데이터 렌더링** 구조 구현
- API 응답 데이터를 화면에 효율적으로 반영하는 설계
- 상태 관리 및 컴포넌트 구조 분리

### 주요 구성 요소

| 레이어 | 역할 |
|---|---|
| **Frontend** | 사용자 인터페이스 구성, API 데이터 기반 렌더링, 컴포넌트 단위 구조 설계 |
| **Backend / Mock API** | 상품 데이터 제공, REST API 형태로 데이터 전달 |

### 데이터 흐름

```
사용자 요청 → API 호출 → 데이터 수신 → 상태 업데이트 → UI 반영
```

> **핵심 원칙:** 단방향 데이터 흐름 유지

## ⚙️ 개발 환경 설정

## 🔥 핵심 개발 포인트

### 1. 컴포넌트 분리 설계
UI를 작은 단위로 쪼개 **재사용성과 유지보수성**을 확보했습니다.  
코드 가독성 향상과 기능 확장이 용이한 구조를 목표로 했습니다.

### 2. API 중심 개발 방식
UI보다 데이터 흐름을 먼저 고려하고, API 응답 구조에 맞춰 컴포넌트를 설계했습니다.

> **"데이터 구조가 UI 구조를 결정한다"**

### 3. 상태 관리 전략
최소한의 상태만 유지하여 **불필요한 리렌더링을 방지**했습니다.

### 4. 리팩터링
중복 코드 제거, 조건문 단순화, 함수 분리를 통해 **유지보수성을 높이고 기술 부채를 줄였습니다.**
